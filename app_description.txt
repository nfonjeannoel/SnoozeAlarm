# Snoozeloo Alarm App - Comprehensive Project Description

## Overview
Snoozeloo is a modern, feature-rich alarm clock application built with Jetpack Compose. The app allows users to create, manage, and customize multiple alarms with features like custom names, ringtones, repeat schedules, and snooze functionality. We'll be implementing the extended version with all features shown in the mockups.

## Design System & Architecture

### Theme & Colors
- Single theme throughout the app (no light/dark mode)
- Primary color: Blue (#3F51B5 or similar to mockups)
- Background: White for screens, light gray for card backgrounds
- Accent colors: Toggle buttons use blue when active
- Text: Black for primary text, gray for secondary information

### Typography
- Use Material Design typography scale with Roboto font
- Large displays for alarm times (prominently displayed)
- Medium text for alarm names and main UI elements
- Small text for secondary information like "Alarm in X hours"

### App Architecture
- MVVM architecture with Compose UI
- Repository pattern for alarm data
- Use Room database for persistent storage
- Alarm Manager for scheduling alarms
- Navigation using Compose Navigation

## Data Models

### Alarm Model
```kotlin
data class Alarm(
    val id: Long = 0,
    val timeHour: Int,
    val timeMinute: Int,
    val name: String = "",
    val isEnabled: Boolean = true,
    val repeatDays: List<DayOfWeek> = emptyList(),
    val ringtone: String = "Default",
    val volume: Float = 0.5f, // 50% default
    val vibrate: Boolean = false
)

enum class DayOfWeek {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}
```

## Screen-by-Screen Implementation

### 1. Splash Screen
- Simple screen with app logo (alarm clock icon on blue background)
- Display for 2-3 seconds before transitioning to Alarm List Screen

### 2. Alarm List Screen
- **Layout**: Column with app title at top, list of alarm cards in ScrollableColumn, FAB at bottom-right
- **Header**: "Your Alarms" text in large font
- **Empty State**: Show when no alarms exist
  - Display message "It's empty! Add the first alarm so you don't miss an important moment!"
  - Show alarm clock icon
  - Ensure FAB is still visible
- **Alarm Cards**: For each alarm display:
  - Optional alarm name at top left (if set)
  - Time in large format (e.g., "10:00 AM")
  - Next occurrence text "Alarm in Xh YYmin"
  - Weekday chips showing which days the alarm repeats on
  - Toggle switch to enable/disable the alarm
  - For extended version: Text showing bedtime suggestion ("Go to bed at XX:YYpm to get 8h of sleep")
- **FAB**: Floating Action Button with plus icon to add new alarm
  - On click, navigate to Alarm Detail Screen in creation mode

### 3. Alarm Detail Screen
- **Layout**: Column with back button, time input, alarm settings cards
- **Time Input Fields**:
  - Two input fields for hours and minutes with colon separator
  - Accept only digits (0-9)
  - Validate for valid 24-hour time format (00:00 to 23:59)
  - Show error if invalid time
  - Text below showing when alarm will next trigger
- **Alarm Name Input**:
  - Card with "Alarm Name" label and current value
  - On tap, show dialog to input name
  - Only save when dialog's "Save" button is tapped
- **Repeat Settings**:
  - Weekday selector with chips for each day (Mon, Tue, Wed, Thu, Fri, Sat, Sun)
  - Selected days have filled background color
- **Ringtone Settings**:
  - Card showing current ringtone selection
  - On tap, navigate to Ringtone Setting screen
- **Volume Settings**:
  - Slider control for volume (0-100%)
  - Default set to 50%
- **Vibration Settings**:
  - Toggle switch for vibration
- **Save Button**:
  - Only enabled when alarm time is valid
  - On tap, save alarm and return to Alarm List

### 4. Ringtone Setting Screen
- **Layout**: Column with back button and list of ringtone options
- **Header**: "Ringtone Setting" text
- **Ringtone List**:
  - Option for "Silent" at top
  - List of default Android ringtones
  - Each item has an icon and name
  - Currently selected ringtone has a checkmark
- **Functionality**:
  - On tap, mark item as selected
  - Play short preview of selected ringtone
  - Do not navigate back automatically after selection

### 5. Alarm Trigger Screen
- **Layout**: Full screen with large time display
- **Content**:
  - Alarm icon at top
  - Large display of current time
  - Alarm name (if set)
  - "Turn off" button to dismiss alarm
  - "Snooze for 5 min" button to snooze alarm
- **Functionality**:
  - Show when an alarm is triggered
  - Play selected ringtone at set volume
  - Vibrate if enabled
  - "Turn off" dismisses screen and stops sound
  - "Snooze" stops alarm and schedules new one for 5 minutes later

## Navigation Flow
- App starts at Splash Screen → Alarm List Screen
- Alarm List Screen → Alarm Detail Screen (when adding new or editing)
- Alarm Detail Screen → Ringtone Setting Screen (when selecting ringtone)
- From any screen → Alarm Trigger Screen (when alarm triggers)

## Functionality Implementation Notes

### Alarm Scheduling
- Use AlarmManager to schedule alarms
- Handle device reboots to reschedule alarms
- Support exact alarm timing for API 31+
- Calculate next occurrence based on:
  - Current time
  - Selected alarm time
  - Repeat days
  - If alarm time is earlier than current time and no repeat days, schedule for next day

### Time Format
- Use 24-hour format internally
- Display in user's preferred format (12/24 hour based on system settings)
- Format "Go to bed" recommendations properly with AM/PM

### Alarm Storage
- Use Room database for persistent storage
- Create appropriate DAOs for CRUD operations
- Implement repository pattern

### Notifications & Foreground Service
- Run foreground service when alarm triggers
- Show notification with alarm info
- Support full screen intent for Alarm Trigger Screen
- Ensure alarm works even when app is closed

### UI Interactions
- Implement smooth animations for transitions
- Use Material Design interaction patterns
- Ensure all interactive elements have proper accessibility support

## Special Features Implementation

### Next Alarm Calculation
- Show "Alarm in Xd Yh Zmin" format
- For repeating alarms, calculate next occurrence based on current day and time
- Update this calculation in real-time

### Bedtime Suggestion
- Calculate 8 hours before alarm time
- Format as "Go to bed at XX:YYpm to get 8h of sleep"
- Update when alarm time changes

### Alarm Sound
- Handle audio focus correctly
- Increase volume gradually
- Support silent mode option
- Implement proper cleanup when alarm dismisses

## Testing Considerations
- Test alarm scheduling across device reboots
- Test time calculations thoroughly, especially edge cases
- Test UI in different screen sizes
- Ensure proper handling of system settings changes
